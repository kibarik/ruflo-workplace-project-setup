# LLM Test Navigator

Instructions for LLM agents working with tests.

## Quick Reference for LLM

### When Implementing a Feature
1. Find existing test: `grep -r "should.*[keyword]" tests/features/`
2. Add test to appropriate feature file
3. Use existing selectors from codebase
4. Run `npm test -- [feature-file].spec.ts`

### When Adding New Feature
1. Create `tests/features/[feature-name].spec.ts`
2. Copy template from below
3. Update `MANIFEST.md`
4. Use only `data-testid` selectors

### Test Template (Copy This)

```typescript
import { test, expect } from '@playwright/test';

/**
 * [Feature Name] Feature Tests
 *
 * Tests:
 * - [capability 1]
 * - [capability 2]
 * - [capability 3]
 */

test.describe('[Feature Name]', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('/path-to-feature');
  });

  test.describe('[Sub-feature Group]', () => {
    test('should [expected behavior]', async ({ page }) => {
      // Arrange
      await [setup];

      // Act
      await [user action];

      // Assert
      await expect(page.locator('[data-testid="result"]'))
        .toBeVisible();
    });

    test('should handle [error case]', async ({ page }) => {
      // Act
      await [action that fails];

      // Assert
      await expect(page.locator('.error-message'))
        .toContainText('[expected error]');
    });
  });
});
```

## Selector Rules

### ✅ Good (LLM-safe)
- `[data-testid="submit-button"]`
- `[data-agent-type="coder"]`
- `[data-status="active"]`
- `[name="email"]`

### ❌ Bad (fragile)
- `.btn-primary` (CSS classes change)
- `#submit-btn-123` (IDs change)
- `button:last-child` (structure changes)

## Common Patterns

### Form Submission
```typescript
await page.fill('[name="field"]', 'value');
await page.click('[data-testid="submit-button"]');
await expect(page.locator('[data-testid="success"]')).toBeVisible();
```

### Async Wait (ALWAYS use waitForSelector)
```typescript
// ✅ Good - waits for specific condition
await page.click('[data-testid="async-action"]');
await expect(page.locator('[data-status="complete"]'))
  .toBeVisible({ timeout: 10000 });

// ❌ Bad - arbitrary timing
await page.waitForTimeout(2000);
```

### Error Handling
```typescript
await page.fill('[name="invalid-field"]', 'invalid-value');
await page.click('[data-testid="submit-button"]');
await expect(page.locator('.error-message'))
  .toContainText('expected error text');
```

## Test File Discovery

```bash
# Find test by keyword (case-insensitive)
grep -ri "swarm" tests/features/

# List all test files
ls tests/features/*.spec.ts

# Run specific test
npm test -- swarm-coordination.spec.ts

# Run tests matching pattern
npm test -- --grep "should initialize"
```

## Checklist Before Adding Test

- [ ] Uses `data-testid` selectors
- [ ] Follows AAA pattern
- [ ] Has clear test name (should ...)
- [ ] Tests single behavior
- [ ] Independent of other tests
- [ ] Cleans up in beforeEach if needed
